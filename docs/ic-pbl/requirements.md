## STEP 1 — 요구사항 분석

### 1.1 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | EDK — Erica Drug King |
| 모듈명 | EDK(Erica drug king) |
| 조 이름 | 안산ic |
| 조원 | 유재준, 송지우, 반영석, 신희찬 |
| 목표 | 증상 선택 기반 일반의약품/영양제 실제 판매 키오스크 |
| 구현 언어 | Python (PyQt6 GUI) |
| 인터페이스 | CLI + PyQt6 GUI (--gui 플래그로 전환) |
| 과목 | 프로그래밍기초 (IC-PBL) |

### 1.2 프로젝트 배경 및 가설

소프트웨어융합대학 학생들은 과제, 프로젝트, 야간 학습 등으로 장시간 건물 내에 머무르는 경우가 많으며,
두통·감기·소화불량·피로 등 가벼운 증상 발생 시 빠르게 필요한 제품을 구매하고자 하는 수요가 존재한다.

**가설**: 증상 선택 기반 무인 판매 키오스크를 제공하면 학생들이 일반의약품 및 영양제를
보다 쉽고 직관적으로 탐색·구매할 수 있으며, 무인 환경에서도 편의성과 접근성이 향상될 것이다.

### 1.3 취급 품목 원칙

| 구분 | 취급 여부 | 예시 |
|------|----------|------|
| 영양제 / 건강기능식품 | ✅ 취급 | 비타민, 오메가3, 아로나민골드, 박카스 |
| 일반의약품 (약국 외 판매 가능) | ✅ 취급 | 타이레놀, 판콜에이, 마데카솔, 게보린 |
| 전문의약품 (처방전 필요) | ❌ 취급 불가 | 항생제, 수면제 등 |
| 한약 / 의료기기 | ❌ 취급 불가 | 별도 허가 필요 |

> 무허가 판매 방지를 위해 관리자가 등록한 품목만 판매 가능하도록 설계한다.

### 1.4 사용자

| 항목 | 내용 |
|------|------|
| 주요 사용자 | 소프트웨어융합대학 기숙사/학관 학생 |
| 역할 구분 | 일반 사용자 / 관리자 |
| 인증 | 관리자 비밀번호 인증 필요 |

## STEP 2 — 확정된 요구사항

### 2.1 핵심 기능 (MVP)

| 기능 | 설명 |
|------|------|
| 증상 선택 | 두통 / 감기 / 소화불량 / 피로 / 외상 등 카테고리 선택 |
| 제품 추천 | 선택한 증상에 해당하는 판매 가능 제품 목록 표시 |
| 상세 정보 | 제품명, 효능, 복용법, 주의사항, 가격, 재고 안내 |
| 장바구니 | 여러 제품 담기, 수량 변경 |
| 결제 | 현금(누적 투입 + 잔돈 반환) + 카드(시뮬레이션) |
| 재고 관리 | 재고 소진 시 선택 불가 처리 |
| 응급 분기 | 응급 증상 감지 시 "즉시 병원 방문" 안내 화면으로 분기 |
| 관리자 기능 | 제품 정보/재고/가격 관리, 현금 보유량 관리, 비밀번호 변경 |

### 2.2 증상 카테고리 및 제품 매핑 (초안)

| 증상 카테고리 | 취급 제품 예시 |
|-------------|--------------|
| 두통 / 열 | 타이레놀(일반의약품), 이지엔6(일반의약품), 게보린(일반의약품) |
| 감기 (코막힘/인후통) | 판콜에이(일반의약품), 화이투벤(일반의약품) |
| 소화불량 / 속쓰림 | 훼스탈골드(일반의약품), 겔포스(일반의약품) |
| 피로 / 영양 보충 | 아로나민골드(영양제), 비타500(영양제), 박카스(영양제) |
| 외상 / 상처 | 마데카솔(일반의약품), 후시딘(일반의약품), 밴드(의료용품) |

### 2.3 화면 흐름

```
[대기 화면]
    ↓ 터치
[증상 선택 화면]
    ↓ 증상 선택         응급 증상 감지 시
    ├──────────────────→ [응급 안내 화면] → [대기 화면]
    ↓
[제품 목록 화면]
    ↓ 제품 선택
[제품 상세 / 장바구니 담기]
    ↓
[장바구니 화면]
    ↓
[결제 수단 선택]
    ├── 현금 → [현금 투입 화면] → [영수증 화면] → [대기 화면]
    └── 카드 → [카드 결제 시뮬레이션] → [영수증 화면] → [대기 화면]
```

### 2.4 기존 코드 재활용 전략

| 기존 구성요소 | EDK에서의 역할 |
|-------------|--------------|
| `Product` (Coffee, Gummy) | `Medicine` — 의약품/영양제 1종 |
| `OptionGroup` / `CustomOption` | `SymptomGroup` / `Symptom` — 증상 카테고리/항목 |
| `Ingredient` (재고) | 의약품/영양제 재고 수량 관리 |
| `Cart` | 구매 장바구니 — 그대로 재활용 |
| `Payment` (CashPayment, CardPayment) | 그대로 재활용 (실제 판매이므로 결제 필요) |
| `ChangeReserve` | 잔돈 관리 — 그대로 재활용 |
| `KioskController` | DrugController로 리팩터링 |
| `DataManager` | 의약품 데이터 JSON 관리 |
| `gui/screens/` | EDK 화면 흐름에 맞게 교체 |

### 2.5 관리자 기능 목록

- 제품 정보 추가 / 수정 / 삭제
- 재고 수량 보충
- 가격 변경
- 제품 판매 ON/OFF
- 현금 보유량 관리 (권종별)
- 관리자 비밀번호 변경
