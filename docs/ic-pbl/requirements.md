## STEP 1 — 요구사항 분석

### 1.1 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | ic-pbl 아이스크림 키오스크 |
| 팀 이름 | 안산ic |
| 팀원 | 유재준, 송지우, 반영석, 신희찬 |
| 목표 | 아이스크림을 완전 커스터마이징할 수 있는 무인 키오스크 |
| 구현 언어 | Python (PyQt6 GUI) |
| 과목 | 프로그래밍기초 (IC-PBL) |

### 1.2 프로젝트 배경 및 제품 선택 근거

인스타그램 등 SNS에서 음식을 단순 소비가 아닌 **자기 표현 수단**으로 사용하는 문화가 확산되고 있다.
*'나는 이런 것을 좋아하는 사람이다'*를 소비로 표현하려면 두 가지가 필요하다:
커스터마이징 가능성, 그리고 결과물이 SNS에 올릴 만한 비주얼이어야 한다는 것.

**후보 검토 과정:**

| 후보 | 탈락 이유 |
|------|----------|
| 의약품 키오스크 | 조합으로 개인 성향 어필 어려움, 커스터마이즈 특성 활용 불가 |
| 커피 키오스크 | 커스터마이징 가능하나 카페 포화 시장, 차별화 근거 부족 |
| **아이스크림 키오스크** | 모양/코팅/토핑/맛 조합 무한, SNS 비주얼, 배스킨라빈스 대비 완전 커스터마이징 제공 |

### 1.3 사용자

| 역할 | 설명 |
|------|------|
| 일반 사용자 | 터치스크린으로 아이스크림 주문 |
| 관리자 | 비밀번호 인증 후 재고/가격/현금/설정 관리 |

## STEP 2 — 확정된 요구사항

### 2.1 핵심 기능 (MVP) — 전체 구현 완료

| 기능 | 설명 |
|------|------|
| 상품 목록 | stick (₩3,000~) / scoop (₩4,000~), 품절 시 선택 불가 |
| 옵션 커스터마이징 | 형태, 코팅, 토핑, 용기, 맛 — 각 옵션별 추가 금액 |
| 실시간 미리보기 | 옵션 선택 시 stick/scoop 미리보기 위젯 즉시 갱신 |
| TTS 음성 안내 | 화면 진입 시 안내 멘트 (edge-tts + pygame) |
| 재고 기반 주문 | 재고 부족 옵션 비활성화, 선택 시 차감 |
| 장바구니 | 여러 상품 담기, 수량 변경, 재고 연동 |
| 현금 결제 | 권종 누적 투입 (1K/5K/10K/50K), 잔돈 반환 (그리디) |
| 카드 결제 | 시뮬레이션 처리 |
| 영수증 | 주문 내역, 결제 수단, 잔돈 표시 |
| 관리자 모드 | PIN (scrypt 해시), 상품 ON/OFF, 가격 변경, 현금 관리, PIN 변경 |
| JSON 영속성 | 키오스크 재시작 후 재고/현금 상태 유지 |

### 2.2 화면 흐름

```
[대기 화면]
    ↓ 터치
[상품 목록]
    ↓ 상품 선택
[옵션 커스터마이징]
    ↓ 장바구니 담기
[장바구니]
    ↓ 결제 진행
[결제 수단 선택]
    ├── 현금 → [현금 투입] → [영수증] → [대기 화면]
    └── 카드 → [카드 결제 시뮬레이션] → [영수증] → [대기 화면]

[대기 화면] → 관리자 버튼
    → [관리자 인증] → [관리자 메뉴] → [대기 화면]
```

### 2.3 관리자 기능 목록

| 영역 | 기능 |
|------|------|
| 재고 관리 | 재료 보충, 재고 현황 확인, 상품 ON/OFF |
| 현금 관리 | 현금통 잔액 확인, 거스름돈 추가 |
| 보안/종료 | 비밀번호 변경 (scrypt 해시), 키오스크 종료 |
