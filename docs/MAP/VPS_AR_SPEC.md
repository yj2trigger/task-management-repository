# VPS·AR 앵커 기능 스펙

> 적용 서비스: MAP 프로젝트 (we-meet-trip)
> 최종 갱신: 2026-06-26 20:46
> 상태: 설계 단계 (미구현)

---

## 개요

여행지에서 AR 앵커(텍스트·포인트)를 장소에 고정시키는 VPS(Visual Positioning System) 기반 기능.
Immersal 방식의 crowd-sourced mapping으로 사전 전체 스캔 부담을 분산.
사용자는 여행 중 1회성으로 가볍게 즐기는 것을 전제로 설계.

---

## 이용 흐름

### 1. 좌표계 선택
- 지도 위에 점으로 기존 좌표계(맵) 목록 표시
- 사용자가 매칭할 좌표계 선택

### 2. 맵 파일 다운로드
- 선택한 좌표계의 맵 파일을 기기에 사전 다운로드
- on-device 방식으로 실시간처럼 동작 (네트워크 지연 제거)

### 3. 부분 스캔 → 로컬 맵 생성 → 서버 맵 매칭
- 사용자가 현재 위치를 부분 스캔
- 로컬 맵 생성 후 다운로드된 서버 맵과 매칭 시도
- **매칭 성공**: 위치 확정, 다음 단계 진행
- **매칭 실패**: 새로운 좌표계 자동 생성 (신규 장소 첫 스캔으로 처리)
- 방식: **Crowd-sourced mapping** — 여러 사용자 스캔을 서버에서 병합해 맵 개선

> **미해결 과제**: 조명 의존성 감소 방법 필요.
> 현재 VPS는 특징점 추출이 조명 변화에 취약. 조명 불변 특징점(예: LiDAR 활용, 야간 보조 조명 힌트) 검토 필요.

### 4. AR 앵커 작성
- 손가락 끝의 상대적 3D 좌표로 공중에 글씨 작성
- VPS 좌표계 기준으로 앵커 위치 저장

### 5. 맵 업데이트 업로드
- 스캔으로 보강된 맵 파일을 서버에 업로드
- 서버에서 기존 맵과 병합 (crowd-sourced 기여)

### 6. 앵커 확인
- 매칭할 좌표계 선택 → 카메라로 해당 부분 비추기
- 위치 매칭 후 앵커 렌더링

## 사용 부담 최소화

| 항목 | 방식 |
|------|------|
| 스캔 분산 | Immersal 방식: 여러 사용자 스캔을 서버에서 합산해 하나의 맵으로 병합 |
| 최초 스캔 | 장소당 최초 1회만 전체 스캔 필요 (이후 사용자는 부분 스캔으로 충분) |
| 다운로드 시점 | 좌표계 선택 직후 백그라운드 다운로드 → 스캔 전 준비 완료 |

---

## 앵커 위치 안내 UI

### 카메라 유도 안내
- 카메라 위치+방향을 UI로 안내
- **한계**: 실내 GPS 정확도 부족으로 초기 방향 안내 정확도 제한
  → 사용자가 카메라를 천천히 회전하며 매칭 트리거 필요

### 미니맵 UI
- 3D 공간 스캔 결과를 2D 평면도로 변환
- 화면 하단 또는 오버레이에 미니맵 표시
- 앵커 위치를 2D 지도 위에 점으로 표시

## 필요 조건 (비기능 요구사항)

| 조건 | 내용 |
|------|------|
| 사용 빈도 가정 | 사용자는 여행 중 재미용으로 **1회만** 사용하는 시나리오 전제 |
| 사전작업 없이 동작 | 누군가 사전 스캔을 하지 않은 장소에서도 이용 가능해야 함 → 첫 사용자가 새 좌표계를 생성하는 흐름으로 해결 |
| 조명 독립성 | 낮/밤, 실내/실외 조건 변화에도 안정적인 매칭 필요 (미해결) |
| 온디바이스 처리 | 매칭 연산은 기기에서 수행, 네트워크 왕복 없이 실시간 동작 |

## 미해결 과제

| # | 과제 | 비고 |
|---|------|------|
| 1 | 조명 의존성 감소 방법 | LiDAR, depth sensor, 조명 불변 descriptor 검토 필요 |
| 2 | 첫 사용자 전체 스캔 UX | 스캔 거부 시 기능 불가 → 온보딩 설계 필요 |
| 3 | 맵 병합 품질 보증 | 저품질 스캔 기여 시 맵 degradation 방지 전략 필요 |
| 4 | 실내 GPS 대안 | 초기 좌표계 탐색을 GPS 없이 수행하는 방법 (QR 코드, BLE beacon 등) |

## 관련 기술 레퍼런스

- **Immersal SDK**: cloud-based VPS, crowd-sourced mapping API
- **ARKit / ARCore**: 기기 측 AR 세션, 특징점 추출
- **VPS (Visual Positioning System)**: 이미지 기반 절대 위치 결정
