# 문제 정의: 기숙사 세탁기 예약 시스템

## 배경

한양대학교 기숙사 세탁기는 선착순 방식으로 운영된다.
사용 가능한 세탁기가 있는지 확인하려면 **직접 세탁실에 가야 한다.**

---

## 현재의 문제

### 1. 정보 비대칭

사용자는 세탁기 현황을 알 수 없다.
→ 세탁실에 갔더니 모두 사용 중인 경우 허탕.
→ 언제 비는지 알 방법 없음 → 반복 방문 또는 포기.

### 2. 대기 방식의 비효율

세탁기가 다 차 있을 때 대기하려면 세탁실 앞에서 기다려야 한다.
→ 시간 낭비, 기숙사 생활 패턴 방해.

### 3. 성별 구역 혼재

기숙사 세탁기는 남/여 구역이 분리됨.
→ 현황 조회와 예약도 성별별로 분리되어야 함.

---

## 해결 목표

| 문제 | 해결 목표 |
|------|----------|
| 세탁기 현황 불투명 | 실시간 세탁기 상태(사용 가능 / 사용 중 / 예약됨) 조회 |
| 대기를 위한 현장 체류 | 원격 대기열 등록 + 빈 세탁기 발생 시 앱 알림 |
| 성별 구역 혼재 | 성별별 채널 분리 (WS, 예약, 조회) |
| 비학교 구성원 사용 방지 | @hanyang.ac.kr 이메일 인증 |

---

## 사용 시나리오

### Mode A — 바로 사용 (여유 있음)

```
사용 가능한 세탁기 多 → 사용자가 직접 층/번호 선택 → 즉시 예약
```

### Mode B — 소프트 예약 (경쟁 상황)

```
사용 가능한 세탁기 少 → 시스템이 배정 → 10분 내 미사용 시 자동 해제
```

### Mode C — 대기열 (만원 상태)

```
사용 가능한 세탁기 없음 → 대기열 등록
→ 세탁기 빌 때 첫 번째 대기자에게 알림
→ 알림 받은 사용자 → Mode B로 전환
```

## 사용자

- **일반 사용자:** 한양대 기숙사 재학생 (@hanyang.ac.kr 이메일 보유)
- **관리자:** 기숙사 담당자 — 세탁기 상태 수동 관리, 기기 추가/제거
- **IoT 장치 (예정):** 세탁기에 부착된 센서 — 작동 상태를 서버에 자동 전송

## 핵심 제약

| 제약 | 내용 |
|------|------|
| 실시간성 | 세탁기 상태 변경 시 즉시 모든 사용자에게 반영 |
| 동시성 | 같은 세탁기를 두 사람이 동시에 예약하면 안 됨 |
| 오프라인 안전 | IoT 장치 없어도 어드민 수동 조작으로 운영 가능 |
| 접근 제한 | 한양대 학생만 사용 가능 |
